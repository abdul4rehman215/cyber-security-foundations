Q1. What is the purpose of using curl in web testing?  
A. It sends HTTP requests and analyzes server responses from the CLI.

Q2. What does a basic GET request do?  
A. Retrieves data without modifying server state.

Q3. Why use curl -v?  
A. To inspect full HTTP request and response headers.

Q4. How are parameters passed in GET requests?  
A. Via URL query strings or -G with -d.

Q5. Difference between GET and POST?  
A. GET retrieves data; POST sends data.

Q6. Why is JSON used in APIs?  
A. It is lightweight, readable, and machine-friendly.

Q7. What do HTTP status codes indicate?  
A. Success or failure of a request.

Q8. Why is error handling important in API clients?  
A. Prevents crashes and ensures resilience.

Q9. How does Python help in API automation?  
A. Simplifies request handling and response parsing.

Q10. Why are APIs critical in cybersecurity?  
A. They enable automation, intelligence feeds, and integrations.
